# 现货 vs 合约(2-5x) 深度对比分析 — HotHunter 策略

## 1. 核心差异一览

| 维度 | 现货 | 合约 2x | 合约 5x |
|------|------|---------|---------|
| 500U 最大仓位 | 500U | 1000U | 2500U |
| 做空能力 | ❌ 只能做多 | ✅ 可做多+做空 | ✅ 可做多+做空 |
| 单笔-8% 亏损 | 亏 40U (8%) | 亏 80U (16%) | 亏 200U (40%) |
| 爆仓风险 | 无（币归零除外） | 极低（需跌50%+） | 低（需跌20%） |
| 资金费率 | 无 | 有（8h/次，~0.01%） | 有（8h/次，~0.01%） |
| 手续费 | 0.1% (BNB折扣后~0.075%) | 0.04%/0.06% maker/taker | 0.04%/0.06% maker/taker |
| 资金利用率 | 100% | 50%（2x可用一半保证金） | 20%（5x可用更少保证金） |
| 适合持仓周期 | 中长期 | 中短期 | 短期/超短线 |

## 2. 针对 HotHunter 热点追踪策略的适配分析

### 2.1 热点追踪的交易特征

```
热点币生命周期（典型）：
  
  T+0   发现信号 → 入场
  T+2h  热度攀升 → 浮盈中
  T+6h  达到高潮 → 分批止盈
  T+12h 热度消退 → 全平
  
  平均持仓: 2~12 小时
```

**关键特征：持仓时间短、胜率中等(40-55%)、盈亏比高(1:2~1:3)**

### 2.2 现货优势

| 优势 | 说明 |
|------|------|
| 🛡️ 零爆仓风险 | 不会被强平，最差是币归零（小币种风险） |
| 💤 睡得着觉 | 不需要半夜盯盘担心爆仓 |
| 📉 回撤可控 | 最大亏损 = 买入成本 - 当前市价 |
| 🧹 简单干净 | 无需管理保证金、无需担心资金费率侵蚀 |
| 📊 适合复利 | 每次盈利都是"真钱"，可以直接复投 |

### 2.3 现货劣势

| 劣势 | 说明 |
|------|------|
| 🐢 增长速度慢 | 500U做现货，即使抓到+25%的热点，也只赚125U |
| 🔒 只能做多 | 熊市/回调时只能空仓等待，无法从下跌中获利 |
| 💰 资金效率低 | 500U只能买500U的币 |

### 2.4 合约 2-5x 优势

| 优势 | 说明 |
|------|------|
| 🚀 加速增长 | 500U × 2x = 1000U 仓位，+25% = +250U（vs 现货125U） |
| 📉 可做空 | 热点消退时可以反手做空，双向盈利 |
| 💰 资金效率高 | 开1000U仓位只需500U保证金，剩余500U可做其他用途 |
| 🎯 小资金友好 | 100U起步时，2x杠杆让仓位规模更"可操作" |

### 2.5 合约 2-5x 劣势（⚠️ 关键风险）

| 劣势 | 说明 |
|------|------|
| 💀 爆仓风险 | 5x杠杆跌20%即爆仓，热点币波动极大（日内±30%常见） |
| 💸 资金费率侵蚀 | 持仓超过8小时，每8h扣0.01%，长期复利下来可观 |
| 😰 心理压力 | 杠杆放大盈亏波动，500U→浮亏200U（40%）仅需价格跌8% |
| 🔧 复杂度增加 | 需要管理保证金率、标记价格、强平价格 |
| 📉 回撤放大 | 原策略"28万U→1万U"如果用杠杆，可能直接归零 |

## 4. 综合结论与推荐

### 4.1 评分矩阵

| 评估维度 | 现货 | 2x合约 | 5x合约 | 权重 |
|----------|------|--------|--------|------|
| 长期稳定性 | ⭐⭐⭐⭐⭐ | ⭐⭐⭐ | ⭐ | 30% |
| 收益潜力 | ⭐⭐ | ⭐⭐⭐⭐ | ⭐⭐⭐⭐⭐ | 20% |
| 回撤控制 | ⭐⭐⭐⭐⭐ | ⭐⭐⭐ | ⭐ | 25% |
| 心理舒适度 | ⭐⭐⭐⭐⭐ | ⭐⭐⭐ | ⭐ | 10% |
| 运维复杂度 | ⭐⭐⭐⭐⭐ | ⭐⭐⭐ | ⭐⭐ | 10% |
| 2G2核适配 | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐ | ⭐⭐⭐ | 5% |
| **加权总分** | **4.4** | **3.2** | **1.8** | 100% |

### 4.2 推荐方案：现货为主 + 可选低杠杆增强

```
┌────────────────────────────────────────────┐
│            推荐：分层策略                    │
│                                             │
│  核心层 (70%资金) → 现货热点追踪             │
│  ├── 稳健复利，长期持有                       │
│  ├── 利润阶梯锁定                             │
│  └── 目标：年化 100-300%                     │
│                                             │
│  增强层 (30%资金) → 2x合约（可选）            │
│  ├── 仅在确认强势趋势时使用                    │
│  ├── 严格止损 -5%（价格层面，即杠杆后-10%）    │
│  ├── 不隔夜持仓（避免资金费率）                │
│  └── 目标：年化 200-500%（但风险更高）         │
│                                             │
│  保险层 → 利润提取                             │
│  ├── 每达到一个资产里程碑，提取10-20%利润      │
│  └── 提取的利润不再参与任何风险交易             │
└────────────────────────────────────────────┘
```

### 4.3 最终建议

> **先用现货跑 3-6 个月**，积累实盘数据和信心。如果现货策略稳定盈利（月均+10%以上，最大回撤<20%），再考虑用利润的一部分（非本金）尝试 2x 低杠杆合约增强。

**为什么？**
1. 你提到的案例从 100U→28万U 用了 40 天，但99%的情况下这是运气+高杠杆的产物，不可复制
2. 长期稳定盈利的核心是**活下来**，不是短期暴富
3. 现货让你有充足的时间验证策略逻辑，而不是被杠杆放大错误
4. 500U 现货如果做到月均 +12%，1 年后约 1900U（3.8x），2 年后约 7200U（14x）——这已经是顶级表现了
5. `28万U → 1万U` 的本质原因不是策略不好，而是**没有保护利润的机制**——这个问题在现货中更容易解决

## 5. 对当前架构文档的调整建议

基于以上分析，建议在 [`plans/architecture.md`](plans/architecture.md) 中增加：

- 在"利润阶梯锁定"中明确**利润提取策略**（达到里程碑后提取利润到独立账户）
- 增加 `max_drawdown_recovery` 机制：回撤超过20%后，强制降低仓位50%直到创出新高
- 可选：预留合约增强层的配置接口（但初期不实现）
