# 插针防护、加仓补仓减仓、止盈止损 — 深度设计

## 1. 插针问题分析与防护

### 1.1 热点币插针有多常见？

热点币（尤其是小市值、新上币）的典型K线：

```
  价格
   │     ╱╲                              ← 上影线插针（假突破）
   │    ╱  ╲    ╱╲
   │   ╱    ╲  ╱  ╲  ╱╲
   │  ╱      ╲╱    ╲╱  ╲    ╱╲
   │ ╱                  ╲  ╱  ╲
   │╱                    ╲╱    ╲
   │                            ╲╱  ← 下影线插针（假跌破）
   └─────────────────────────────────> 时间
```

**统计数据（币安小市值现货，5m K线）**：
- 上影线 > 实体 2倍：出现频率 ~35%
- 下影线 > 实体 2倍：出现频率 ~28%
- 单根K线振幅 > 5%：出现频率 ~15%

> 如果仅用收盘价计算指标，约 30-40% 的信号可能是插针噪音。

### 1.2 原方案为什么容易被插针

```
原方案入口信号依赖：
  ROC(6)  ←── 基于收盘价，一根大针直接触发
  RSI(14) ←── 同样基于收盘价
  Vol     ←── 插针往往伴随放量（假量）
  
问题场景：
  T+0  价格 1.00，正常
  T+1  插针到 1.08，收盘 1.02
       → ROC(6) 暴涨 → RSI 飙升 → Vol 放大
       → 热点评分 = 85 → 触发入场！
       → 但实际价格已回落到 1.02，入场即浮亏
```

### 1.3 五层插针防护体系

```
┌─────────────────────────────────────────────────────┐
│               插针防护五层过滤                         │
│                                                      │
│  第1层: 价格源选择                                    │
│  ├── 使用 HLC3 = (High+Low+Close)/3 替代收盘价       │
│  └── 减少单点价格的噪音                                │
│                                                      │
│  第2层: 影线过滤器                                    │
│  ├── 计算上影线比例 = (High - Max(Open,Close)) / Range │
│  ├── 计算下影线比例 = (Min(Open,Close) - Low) / Range │
│  └── 影线 > 60% 的K线 → 标记为"疑似插针"，降权处理    │
│                                                      │
│  第3层: K线实体确认                                    │
│  ├── 阳线实体 > 影线的 50%（即不是纯针）               │
│  └── 最近3根K线中至少有2根满足此条件                   │
│                                                      │
│  第4层: 多时间框架确认                                 │
│  ├── 5m 信号触发后，检查 15m 趋势                      │
│  ├── 15m EMA9 > EMA21 且 ADX > 20                    │
│  └── 如果15m不确认 → 降低热点评分 20 分               │
│                                                      │
│  第5层: 延迟入场（可选，更保守）                        │
│  ├── 信号触发后，等待下一根K线收盘确认                  │
│  └── 如果下一根K线收盘价 > 信号K线收盘价 → 确认入场     │
│      └── 如果下一根K线回落 → 信号作废                  │
└─────────────────────────────────────────────────────┘
```

### 1.4 修正后的热点识别指标

```
热点评分 = 动量分(35%) + 量能分(30%) + 趋势分(25%) + 确认分(10%) - 插针罚分

动量分 (35%):
  - ROC(6) 使用 HLC3 计算
  - ROC(12) 使用 HLC3 计算
  - 两根K线的收盘价位置（在K线实体的中上部 = 强势）

量能分 (30%):
  - 当前成交量 / SMA(成交量, 20) 
  - MFI(14) 资金流量指数（已内置高/低/收，天然抗针）
  - OBV 趋势（累积/派发判断，比单根量更可靠）

趋势分 (25%):
  - EMA(9) > EMA(21) — 使用 HLC3
  - ADX(14) > 20（从25降到20，因为确认分补充了可靠性）

确认分 (10%):  ← 新增
  - 15m 时间框架 EMA9 > EMA21（5分）
  - 最近3根K线实体确认（3分）
  - K线收盘在实体的上1/3位置（2分）

插针罚分:  ← 新增
  - 当前K线上影线 > 60%: -15分
  - 前一根K线上影线 > 60%: -10分
  - 当前K线下影线 > 60% 且收阳: -5分（可能是支撑确认，罚分减半）
  - 连续2根K线都有大针: 额外 -10分
```

### 1.5 修正后的入场条件

| 条件 | 原参数 | 修正后 | 说明 |
|------|--------|--------|------|
| 热点评分 | ≥ 70 | ≥ 65 | 降低阈值（因为插针罚分更严格） |
| 插针罚分 | 无 | ≥ -25 | 罚分不超过-25（太差直接过滤） |
| 成交量爆发 | Vol > SMA20×1.5 | Vol > SMA20×1.5 且 OBV向上 | 增加OBV确认 |
| 趋势确认 | EMA9>EMA21, ADX>25 | 5m: EMA9>EMA21, ADX>20 + 15m: EMA9>EMA21 | 多时间框架 |
| RSI范围 | 55<RSI<78 | 50<RSI<75 | 放宽下限，收紧上限 |
| 价格位置 | Close > EMA21 | HLC3 > EMA21 | 使用HLC3 |
| **新增** | — | K线实体确认通过 | 防插针 |
| **新增** | — | 距上次入场 > 30min | 防频繁交易 |

## 2. 加仓、补仓、减仓策略

### 2.1 核心原则

| 操作 | 何时用 | 热点追踪是否适用 | 理由 |
|------|--------|:--:|------|
| **补仓** (亏损加仓) | 价格下跌后追加买入摊平成本 | ❌ **不使用** | 热点消退是结构性变化，补仓=接飞刀。原策略28万→1万U的推手之一 |
| **加仓** (盈利加仓) | 趋势确认后追加仓位 | ⚠️ **严格限制** | 仅在趋势极强时有限加仓，有严格前置条件 |
| **减仓** (分批止盈) | 价格上涨后分批卖出 | ✅ **核心策略** | 锁定利润、降低风险暴露 |

### 2.2 补仓策略：明确禁止

```
规则：任何情况下不补仓

原因：
  1. 热点币下跌往往不是"回调"而是"热点转移"
  2. 补仓会让亏损翻倍，违反"单笔风险≤2%"的铁律
  3. 原策略28万U→1万U中，补仓是加速归零的关键因素
  
替代方案：
  如果趋势未破、信号仍在 → 持有现有仓位即可，不加仓
  如果趋势已破 → 止损离场，等待下一个热点
```

### 2.3 加仓策略：金字塔加仓（严格限制条件）

```
触发条件（必须同时满足 全部 5 项）：

  1. 首次入场后浮盈 ≥ +10%（已有足够安全垫）
  2. 15m ADX > 30（趋势极强）
  3. 15m 成交量持续放大（Vol > Vol_SMA20 × 1.3）
  4. 5m 价格回调到 EMA9 附近但未跌破（健康回调）
  5. 总仓位（首次+加仓）≤ 单币仓位上限

加仓规则：
  ┌──────────────────────────────────────────┐
  │  金字塔加仓（逐次缩小）                    │
  │                                           │
  │  首次仓位: 100% 计划仓位                  │
  │  第1次加仓: 50% 计划仓位（总150%）        │
  │  第2次加仓: 25% 计划仓位（总175%）        │
  │  禁止第3次加仓                            │
  │                                           │
  │  加仓后止损线上移：                        │
  │  原始止损: 入场价 × 0.92                  │
  │  加仓后:  平均成本 × 0.95（更紧的止损）    │
  └──────────────────────────────────────────┘

代码实现：
  - 通过 custom_stake_amount 识别是否是加仓（检查是否已有该交易对的持仓）
  - 如果是加仓，限制仓位大小
  - 加仓后的止损通过 custom_stoploss 动态调整
```

### 2.4 减仓策略：分批止盈（核心）

```
┌─────────────────────────────────────────────────────┐
│              分批止盈三阶段                           │
│                                                      │
│  入场价: $1.00   计划仓位: 100U                       │
│                                                      │
│  ═══ 第一阶段: +8% ═══                               │
│  ├── 价格: $1.08                                     │
│  ├── 卖出: 30% 仓位 (30U → 32.4U)                    │
│  ├── 锁定利润: +2.4U                                 │
│  └── 剩余仓位: 70U                                   │
│                                                      │
│  ═══ 第二阶段: +15% ═══                              │
│  ├── 价格: $1.15                                     │
│  ├── 卖出: 再卖 30% 仓位 (21U → 24.15U)              │
│  ├── 锁定利润: +5.55U                                │
│  └── 剩余仓位: 49U                                   │
│                                                      │
│  ═══ 第三阶段: +25% 或 移动止损 ═══                   │
│  ├── 价格: $1.25 → 卖出剩余 40% (49U → 61.25U)       │
│  │        或                                         │
│  ├── 从最高点回撤 ≥ 5% → 移动止损触发全平             │
│  └── 总利润: ~18U (18% 回报，但最大风险已逐级降低)      │
└─────────────────────────────────────────────────────┘

关键设计：
  1. 每批止盈后，移动止损自动跟进到新的基准价
  2. 第一批止盈后，即使价格归零，总体仍盈利
  3. 第三批不设固定价，用移动止损保护
```

## 3. 止盈止损详细逻辑

### 3.1 完整离场决策树

```
持仓中
  │
  ├── [最高优先级] 硬止损检查
  │   └── 当前价 ≤ 入场价 × 0.92
  │       └── 立即市价全平，不等待任何条件
  │
  ├── [次高优先级] 分批止盈检查
  │   ├── 已平第1批? ──否──> 当前价 ≥ 入场价 × 1.08 → 卖出30%
  │   ├── 已平第2批? ──否──> 当前价 ≥ 入场价 × 1.15 → 再卖30%
  │   └── 已平第3批? ──否──> 当前价 ≥ 入场价 × 1.25 → 卖剩余40%
  │
  ├── 移动止损检查 (freqtrade内置)
  │   ├── 浮盈 < 3%: 使用初始硬止损 (-8%)
  │   └── 浮盈 ≥ 3%: 激活移动止损 (回撤5%)
  │       └── 从持仓期间最高点回撤 ≥ 5% → 全平
  │
  └── [最低优先级] 热点消退检查
      ├── Vol < Vol_SMA(20) × 0.7（持续缩量）
      ├── RSI(14) < 40（趋势转弱）
      ├── HLC3 下穿 EMA21（中期趋势破位）
      └── 以上任一满足 → 全平剩余仓位
```

### 3.2 止盈价格计算方式

```
止盈价基于入场价，而非当前价：

  入场价 = 实际成交均价（含手续费）
  
  TP1 = 入场价 × 1.08   (第1批, 30%仓位)
  TP2 = 入场价 × 1.15   (第2批, 30%仓位)  
  TP3 = 入场价 × 1.25   (第3批, 40%仓位)

动态调整规则：
  - 如果触发加仓，TP按平均成本重新计算
  - 如果第一批止盈后价格继续涨但不到TP2，移动止损保护剩余70%
```

### 3.3 移动止损具体参数

```
freqtrade 配置：

  "trailing_stop": true,
  "trailing_stop_positive": 0.03,    # 盈利3%后才激活移动止损
  "trailing_stop_positive_offset": 0.01,  # 激活后允许1%波动
  "trailing_only_offset_is_reached": true,  # 只有达到offset后才开始trail

实际行为：
  价格走势: $1.00 → $1.04 (+4%) → $1.02 (+2%) → $1.06 (+6%) → $1.00
  
  $1.00 入场，止损线 = $0.92（硬止损）
  $1.04 价格 > $1.03（3%触发线），移动止损激活
        止损线 = $1.04 × 0.95 = $0.988（从最高点$1.04回撤5%）
  $1.02 价格 > $0.988，继续持有
  $1.06 新最高点，止损线更新 = $1.06 × 0.95 = $1.007
  $1.00 价格 < $1.007 → 触发移动止损 → 全平
  
  结果: 盈利 ~0.7%（几乎保本），避免了从+6%到亏损的过山车
```

### 3.4 分批止盈与加仓：adjust_trade_position（Freqtrade 2026.5.1 原生 API）

✅ Freqtrade 2026.5.1 stable 已原生支持 `adjust_trade_position()` 回调，可直接实现部分平仓和加仓。

**配置要求**：
- `config.json`: `"position_adjustment_enable": true`
- 策略类属性: `position_adjustment_enable = True`

**API 契约**：
- 返回 **正值** → 加仓（买入指定 stake_amount 金额）
- 返回 **负值** → 减仓/部分平仓（卖出绝对值金额）
- 返回 `None` 或 `0` → 不调整

```python
def adjust_trade_position(self, trade, current_time, **kwargs):
    """
    分批止盈 + 金字塔加仓 统一入口
    """
    profit_pct = trade.calc_profit_ratio(current_time)
    
    # --- 分批止盈（返回负值 = 卖出） ---
    if profit_pct >= 0.25 and not self._tp3_done(trade):
        return - (trade.stake_amount * 0.40)   # 卖出40%
    if profit_pct >= 0.15 and not self._tp2_done(trade):
        return - (trade.stake_amount * 0.30)   # 卖出30%
    if profit_pct >= 0.08 and not self._tp1_done(trade):
        return - (trade.stake_amount * 0.30)   # 卖出30%
    
    # --- 金字塔加仓（返回正值 = 买入）---
    if self._should_pyramid(trade, current_time):
        return self._calc_add_amount(trade)     # 买入50%初始仓位
    
    return None  # 不调整
```

**止盈状态追踪**：通过自定义字典 `self._exit_state[trade.id]` 记录每笔交易的已触发止盈批次，防止重复触发。

**移动止损兼容**：`adjust_trade_position` 部分平仓后，Freqtrade 自动更新持仓均价和止损基准，移动止损继续保护剩余仓位。
