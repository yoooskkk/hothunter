# 热点发现机制深度分析 — VolumePairList vs 涨幅榜 vs 全量监控

## 1. 核心问题

当前设计使用 **VolumePairList（成交量Top30）** 作为热点候选池。这引发几个关键问题：

> - 这和币安平台的"热点分类"是什么关系？
> - 和直接看涨幅榜/跌幅榜有什么区别？
> - 哪种方式更适合热点追踪策略？
> - 2G2核服务器上能跑哪种方案？

---

## 2. 四种热点发现路径对比

### 2.1 路径 A：币安平台"热点标签"

```
币安Web/App 提供：
  - "热门" 板块（官方推荐）
  - "涨幅榜"（24h涨幅排行）
  - "新币"（最近上线的币种）
  - "成交量榜"（24h成交量排行）
```

| 维度 | 评价 |
|------|------|
| 获取方式 | 需要调用币安API获取标签/板块数据，或爬虫 |
| 时效性 | **滞后**：标签是T+N更新，不是实时的 |
| 覆盖度 | 仅覆盖被官方标记的币，可能遗漏暗马 |
| API依赖 | 需要额外API（非标准K线API），可能有频率限制 |
| 适用性 | ❌ 不适合自动化交易策略，更适合人工参考 |

### 2.2 路径 B：涨幅榜驱动

```
逻辑: 扫描所有USDT交易对 → 按24h涨跌幅排序 → 取Top N入场
```

| 维度 | 评价 |
|------|------|
| 优势 | 简单直观，直接追涨 |
| 劣势1 | **追高陷阱**：涨了24h的币往往已经到顶部或接近顶部 |
| 劣势2 | **操纵风险**：小币种可以低成本对倒拉盘制造涨幅假象 |
| 劣势3 | **无成交量验证**：可能是无量空涨（庄家对倒） |
| 劣势4 | **无趋势确认**：可能只是超跌反弹，涨一天跌三天 |
| 典型失败 | 看到+50%追进去 → 庄家砸盘 → 套牢 |

### 2.3 路径 C：成交量驱动（当前方案）

```
逻辑: VolumePairList 取成交量Top30 → 策略内指标筛选热点
```

| 维度 | 评价 |
|------|------|
| 优势1 | **真金白银**：成交量难以伪造（需要真实资金） |
| 优势2 | **自动轮换**：VolumePairList每24h自动更新Top30 |
| 优势3 | **Freqtrade原生**：无需额外代码，API消耗低 |
| 优势4 | **量在价先**：成交量放大往往领先于价格爆发 |
| 劣势1 | **滞后性**：成交量排行是24h累计，可能错过爆发初期 |
| 劣势2 | **噪音**：下跌放量也会进入Top30（需要策略内方向过滤） |
| 劣势3 | **遗漏暗马**：新币/小币初期成交量低，进不了Top30 |

### 2.4 路径 D：全量实时监控

```
逻辑: 获取所有USDT交易对(400+) → 每个都计算指标 → 实时筛选热点
```

| 维度 | 评价 |
|------|------|
| 优势 | 不会遗漏任何热点，最全面 |
| 劣势1 | **计算量巨大**：400+交易对 × 每对计算10+指标 = 2G2核难以承受 |
| 劣势2 | **API限制**：币安API权重有限，全量K线拉取可能触发限频 |
| 劣势3 | **过度设计**：违反"单人维护、不过度设计"原则 |
| 适用性 | ❌ 不适合2G2核服务器 |

## 3. 涨幅榜 vs 成交量排行 vs 混合方案

### 3.1 一个真实场景对比

```
场景：某新币 XYZ 上线币安3小时

时间线：
  T+0:00  XYZ上线，价格 $0.10，成交量极低（还进不了Top30）
  T+0:30  第一波拉升到 $0.15 (+50%)，成交量开始放大
  T+1:00  拉到 $0.22 (+120%)，成交量进入币安前30
  T+1:30  价格 $0.28，进入涨幅榜第1名，散户FOMO涌入
  T+2:00  价格 $0.35，VolumePairList 下次刷新时纳入
  T+3:00  价格 $0.25，开始回落
  T+6:00  价格 $0.12，回归起点
```

| 方案 | 发现时间 | 入场价 | 结果 |
|------|----------|--------|------|
| 涨幅榜 | T+1:30 | ~$0.28 | 买入后最高+25%，但大概率被套 |
| VolumePairList | T+2:00 | ~$0.30 (需等下次刷新) | 入场更晚，风险更高 |
| 涨幅榜+Vol验证 | T+1:30 | $0.28 | 同上但有量验证 |
| **混合方案** | T+0:30~1:00 | ~$0.15-0.20 | 早期发现，利润空间最大 |

### 3.2 核心洞察

> **涨幅榜 = 看结果，成交量排行 = 看现象，混合方案 = 看过程**

热点追踪的关键不是"已经涨了很多的币"，而是"正在开始爆发、还有上涨空间的币"。单纯看涨幅榜或成交量排行都太晚了。

## 4. 推荐方案：混合热点发现（修正设计）

### 4.1 三层漏斗

```
┌─────────────────────────────────────────────────────┐
│            混合热点发现 三层漏斗                       │
│                                                      │
│  第1层: StaticPairList（固定关注）                    │
│  ├── BTC/USDT, ETH/USDT, BNB/USDT                   │
│  ├── 目的：稳定收益兜底，不依赖热点                    │
│  └── 数量: ~5个                                      │
│                                                      │
│  第2层: VolumePairList（成交量驱动）                   │
│  ├── 24h成交量 Top 25                                │
│  ├── 自动轮换，Freqtrade原生                         │
│  ├── 目的：捕获已经形成热度的币                        │
│  └── 数量: ~25个                                     │
│                                                      │
│  第3层: 策略内涨幅动量扫描（新增）                     │
│  ├── 对第1+2层的30个币，在策略内计算短期动量           │
│  ├── 5m ROC(6) + 15m ROC(4) 双周期动量               │
│  ├── 目的：在热度形成早期发现信号                      │
│  └── 这是"涨幅"在策略内的正确使用方式                   │
│      （不是看24h涨幅榜，而是看短期动量爆发）            │
└─────────────────────────────────────────────────────┘
```

### 4.2 为什么不是纯粹的涨幅榜？

```
涨幅榜（24h涨幅Top N）的问题：

  假设一个币24h涨了80%：
  
  情况A: 前23h缓慢涨了10%，最后1h暴涨70%
         → 涨幅榜显示+80%，但最后1h才真正爆发
         → 如果我们在涨幅榜看到它时是T+23h，入场后只剩鱼尾
  
  情况B: 开盘暴涨80%，然后横盘23h
         → 涨幅榜显示+80%，但已无上涨动能
         → 入场后被横盘消耗时间成本
  
  情况C: 均匀上涨80%
         → 涨幅榜显示+80%，但什么时候入场？
         → 没有任何入场时机信息

涨幅榜只告诉你"过去发生了什么"，不告诉你"现在正在发生什么"。
```

### 4.3 策略内短期动量扫描（正确使用"涨幅"）

```
不在 Pairlist 层面按涨幅排序，而是在策略 indicators 中计算短期动量：

  ROC_5m_6   = 过去30分钟的价格变化率（捕捉即时爆发）
  ROC_15m_4  = 过去60分钟的价格变化率（确认持续性）
  
  热点信号 = ROC_5m_6 > 3% AND ROC_15m_4 > 5% AND Vol > SMA20×1.5
  
  这等价于："最近30分钟开始拉升 + 最近1小时持续拉升 + 有真量配合"
  
  和24h涨幅榜的本质区别：
  ┌──────────────┬──────────────────┬──────────────────┐
  │              │  24h涨幅榜       │  短期动量扫描     │
  ├──────────────┼──────────────────┼──────────────────┤
  │ 时间窗口     │ 24小时           │ 30-60分钟        │
  │ 发现阶段     │ 已经涨完了       │ 正在涨           │
  │ 入场时机     │ 不明确           │ 精确到5m K线     │
  │ 假信号风险   │ 高（老庄出货）   │ 中（但有量验证） │
  │ 错过风险     │ 低               │ 中（需持续监控） │
  │ 计算成本     │ 低               │ 低（仅30个币）   │
  └──────────────┴──────────────────┴──────────────────┘
```

## 5. 最终推荐：修正后的 Pairlist 配置

### 5.1 配置

```json
{
  "pairlists": [
    {
      "method": "StaticPairList",
      "allow_inactive": false
    },
    {
      "method": "VolumePairList",
      "number_assets": 25,
      "sort_key": "quoteVolume",
      "min_value": 100000,
      "refresh_period": 86400
    }
  ],
  "exchange": {
    "pair_whitelist": [
      "BTC/USDT",
      "ETH/USDT",
      "BNB/USDT",
      "SOL/USDT",
      "DOGE/USDT"
    ]
  }
}
```

### 5.2 策略内的热点筛选流程

```
每根5m K线到来时（仅对~30个候选币执行）：

  第1步：更新插针检测
  ├── 计算影线比例
  ├── 标记插针K线
  └── 计算插针罚分

  第2步：计算短期动量
  ├── ROC(6, 5m)  = (HLC3当前 - HLC3_6根前) / HLC3_6根前
  ├── ROC(4, 15m) = (HLC3当前 - HLC3_4根前) / HLC3_4根前
  └── 动量分 = (ROC_5m 标准化 + ROC_15m 标准化) / 2

  第3步：量能验证
  ├── Vol_Ratio = 当前vol / SMA(vol, 20)
  ├── OBV趋势 = OBV当前 vs OBV_5根前
  └── 量能分 = ...

  第4步：趋势确认
  ├── EMA9 > EMA21 (HLC3)
  ├── ADX(14) > 20
  └── 趋势分 = ...

  第5步：综合评分 + 入场决策
  └── 热点评分 ≥ 65 → 生成入场信号
```

### 5.3 2G2核性能评估

```
候选池: ~30个交易对
每根K线(5m): 对30个币各计算 ~12个指标
TA-Lib加速: 每个指标 ~0.1ms
总计算: 30 × 12 × 0.1ms ≈ 36ms（每5分钟）

内存: 30个币 × 500根K线 × ~2KB ≈ 30MB
总计内存: Freqtrade(~300MB) + 策略数据(~30MB) + Python(~200MB) ≈ 530MB

结论: 2G2核完全够用，CPU占用 < 10%
```

## 6. 总结对比

| 方案 | 发现时机 | 假信号 | 计算成本 | 2G2核 | 推荐 |
|------|:--:|:--:|:--:|:--:|:--:|
| 币安热点标签 | 晚 | 低 | 低 | ✅ | ❌ 不可靠 |
| 24h涨幅榜 | 晚 | 高 | 中 | ✅ | ❌ 追高陷阱 |
| 24h跌幅榜抄底 | — | 极高 | 中 | ✅ | ❌ 接飞刀 |
| VolumePairList only | 中 | 中 | 低 | ✅ | ⚠️ 不够灵敏 |
| **混合方案（推荐）** | **早** | **低** | **低** | **✅** | **✅ 最优** |

### 混合方案 = VolumePairList + StaticPairList + 策略内短期动量扫描

> 核心思想：**不在 Pairlist 层面做涨幅筛选，而是在策略 indicators 中计算短期动量来发现"正在爆发"的币。** VolumePairList 提供"有量"的候选池，策略内的短期动量提供"正在涨"的精确时机。
